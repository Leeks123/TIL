# 스코프와 존재

  • **스코프(=가시성)**는 프로그램의 현재 실행 중인 부분, 실행 컨텍스트에서 현재 보이고 접근할 수 있는 식별자
  • 존재 한다는 것은 식별자가 메모리가 할당된 무언가를 가리킨다는 뜻.. 존재하지만 스코프 안에는 없는 변수가 있을 수 있음

# 변수 숨기기

{
  // 외부 블록
  let x = 'red';
  console.log(x); // "red"
  {
    // 내부 블록
    let x = 3;
    console.log(x); // 3
  }
  console.log(x); // "red"
}
console.log(typedof x); // "undefined"

외부 블록에서의 변수 x 는 내부 블록에서 정의된 새로운 x 에 의해 숨겨진다. 실행흐름이 내부 블록에 들어가 새 변수 x 를 정의하는 순간, 두 변수가 모두 스코프 안에 있다. 그러나 외부 블록의 변수는 접근할 방법이 없다.

# 클로저

클로저 : 함수가 특정 스코프에 접근할 수 있도록 의도적으로 그 스코프에 정의하는 경우

let globalFunc;
{
  let blockVar = 'a';
  globalFunc = function(){
    console.log(blockVar);
  }
}
globalFunc(); // "a"

globalFunc 는 블록 안에서 값을 할당받았다. 이 블록 스코프와 그 부모인 전역 스코프가 클로저를 형성하여, 함수를 어디에서 호출하든, 클로저에 들어 있는 변수에 접근할 수 있다.

# 함수 스코프와 호이스팅

# 함수 스코프

var 변수를 프로그램의 가장 바깥에서 정의하면 전역변수, 이외에 함수 내부에서 정의된 var 변수의 스코프는 함수 내부이다.

함수 안에서 var 키워드를 사용하지 않고 변수할당을 하면 그 변수는 전역변수가 된다.

for, while, switch, if 등의 코드에서 사용한 var 변수도 계속 남아 있게되어 변수의 스코프 문제가 발생할 수 있다.

# 호이스팅

var 로 정의된 변수는 그 변수가 속한 스코프의 최상단으로 끌어올려진다.

console.log(n); // undefined
var n = 1;

위의 코드처럼 호이스팅되어서 정의는 끌어올려지지만 값 할당까지는 이뤄지지 않는다.

const, let 은 블록 스코프이다.

호이스팅도 가능하다. 하지만 변수를 정의하기 전에 그 변수를 사용하려고 하면 참조에러를 발생시킨다.

console.log(n2); // 참조 에러
const n2 = 1;

const 로 정의된 변수는 재할당이 불가능하다. const 로 정의된 변수를 재할당하면 에러가 발생한다.

const 로 정의된 객체의 내부 값의 수정은 가능하다는 점은 주의

var x = 3;
if(x===3){
  var x = 2
  console.log(x); // 2
}
console.log(x); //  2

var 를 이용해 변수를 선언하면 자바스크립트는 같은 변수를 여러번 정의하더라도 무시한다. if 문에서 선언된 x 는 새롭게 정의된 변수가 아니라 바깥 에서 선언된 x 이다.
