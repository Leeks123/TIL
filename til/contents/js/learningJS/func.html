# Function

# this 키워드

메서드를 호출하면 this 는 호출한 메서드를 소유하는 객체

const o = {
  name : "tom",
  speak() { return `My name is ${this.name}`;},
}
o.speak(); // "My name is tom"

const speak = o.speak;
speak(); // "My name is undefined"

this 는 함수를 어떻게 호출했느냐에 따라 달라짐

const o = {
  name : 'tom',
  speak() {
    function greet(){
      return `Hello. my name is ${this.name}`;
    }
    return greet();
  }
}
o.speak() // "Hello. my name is "

const o2 = {
  name : 'tom',
  speak() {
    const self = this;
    function greet(){
      return `Hello. my name is ${self.name}`;
    }
    return greet();
  }
}
o2.speak() // "Hello. my name is tom"

this 가 다른 곳에 묶이는 것을 방지하기 위해 self 변수에 할당하여 사용

# 화살표 함수

화살표 함수와 일반 함수의 차이

  • this 가 다른 변수와 마찬 가지로 정적으로 묶임
const o = {
 name : 'tom',
 speak() {
   const greet = ()=>`Hello. my name is ${this.name}`;

   return greet();
 }
o.speak(); // "Hello. my name is tom"
  • 객체 생성자로 사용할 수 없음
  • arguments 변수도 사용할 수 없음 => 확산 연산자로 대처가능

# call , apply , bind

# call

const tom = { name : "Tom" };
const sam = { name : "Sam" };

function greet() {
  return `Hello. my name is ${this.name}`;
}

greet();	 // "Hello. my name is "
greet.call(tom); // "Hello. my name is Tom"
greet.call(sam); // "Hello. my name is Sam"

call 메서드를 이용해 this 를 특정 값으로 지정할 수 있음. 함수를 호출하면서 call 를 사용할 때 첫번째 이후의 매개변수는 함수의 매개변수로 전달됨

# apply

매개변수를 배열로 받는다는 것을 제외하면 call 과 완전히 같음 es6의 확산연산자(...)와 call의 조합은 apply와 동일하다

# bind

bind 를 사용하면 함수의 this 값을 영구히 바꿀 수 있음. call, apply 를 사용해 호출하더라도 binding된 객체는 변하지 않음

# 순수함수

  • 입력이 같으면 결과도 반드시 같은 함수가 순수함수이다
  • 순수함수에는 부수효과(side Effect)가 없다. 함수를 호출한다고 해서 프로그램의 상태가 바뀌지 않음

코드를 테스트하기 쉽고, 이해하기 쉽고, 재사용하기도 쉬우니까 순수함수로 작성하는 습관을 들이자!!
