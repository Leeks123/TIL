# 객체 지향과 프로토타입

자바스크립트의 프로토타입은 객체의 타입을 정의하는데 편하게 사용하는 수단이지만, 실제로는 함수의 기능이다.

# 인스턴스 생성과 프로토타입

# 객체 인스턴스 생성

  • 객체 청사진으로서의 프로토타입
function Ninja() {}
Ninja.prototype.swingSword = function(){
  return true;
};

var ninja1 = Ninja();
var ninja2 = new Ninja();

console.log(ninja1); // undefined
console.log(ninja2); // Ninja {}
console.log(ninja2.swingSword()); // true

생성자 함수의 prototype 프로퍼티에 swingSword() 메서드를 추가하면, swingSword() 메서드를 간단히 새로 만들어진 객체에 덧불일 수 있다.

  • 인스턴스 프로퍼티
function Ninja() {
  // 인스턴스 메서드와 변수
  this.swung = false;
  this.swingSword = function(){
		return !this.swung;
  };
}

Ninja.prototype.swingSword = function(){
  return this.swung; // 프로토타입 메서드
}

var ninja = new Ninja();
console.log(ninja.swingSword()); // true

생성자 내에서 만들어진 인스턴스 멤버가 프로토타입에 정의된 같은 이름의 프로퍼티를 재정의!

프로토타입의 프로퍼티들이 새로 만들어진 객체의 인스턴스와 바인딩되고, 그 후에 생성자 함수 내에서 정의한 프로퍼티들이 객체 인스턴스에 추가된다.

  • 참조 조정
function Ninja() {
  this.swung = true;
}

var ninja = new Ninja();

Ninja.prototype.swingSword = function(){
  return this.swung;
}

console.log(ninja.swingSword()); // true

프로토타입의 프로퍼티들은 어디에도 복사되지 않고, 그저 생성된 객체에 덧붙일 뿐.. 객체의 어떤 프로퍼티를 참조할 때 해당 객체가 그 프로퍼티를 직접 소유한게 아니라면, 프로토타입에서 그 프로퍼티를 찾음

# 생성자와 객체 타입

function Ninja() {}

var ninja = new Ninja();

console.log(typeof ninja) // object
console.log(ninja instanceof Ninja); // true
console.log(ninja.constructor == Ninja); // true

var ninja2 = ninja.constructor();

console.log(ninja2 == ninja); // false

Ninja 생성자 함수에 직접 접근하지 않고도 ninja 인스턴스를 통해서 ninja2 를 생성했다. 이는 원본 생성자 함수가 더 이상 유효범위에 있지 않더라도 사용할 수 있다.

# 상속과 프로토타입 체인

function Person(){}
Person.prototype.dance = function() {}

function Ninja(){}
Ninja.prototype.dance = { dance: Person.prototype.dance };

var ninja = new Ninja();
console.log(ninja instanceof Ninja); // true
console.log(ninja instanceof Person); // false
console.log(ninja instanceof Object); // true

위의 코드로 상속을 기대하지만 실제로는 복사만 일어났을 뿐이다.

function Person(){}
Person.prototype.dance = function() {}

function Ninja(){}

Ninja.prototype = new Person();
var ninja = new Ninja();

console.log(ninja instanceof Ninja);  // true
console.log(ninja instanceof Person); // true
console.log(ninja instanceof Object); // true

# 실수하기 쉬운 것들

  1. 객체 확장하기
    Object.prototype을 확장하면 모든 객체가 추가된 프로퍼티를 받게 된다.
  2. Number 객체 확장하기
    자바스크립트 엔진이 숫자와 숫자 객체의 프로퍼티를 파싱하는 방법 때문에 혼란 초래할 수 있음
  3. 인스턴스 생성 이슈
    함수 호출과 생성자 호출의 차이로 인한 에러가 발생할 수 있다. 잘못된 방식의 호출에 대한 대비 코드가 필요하다.
