# Interface

# 객체 타입 정의하기

interface Person {
  readonly name: string;
  age: number;
  job?: string;
}
const p1: Person = { name:'Tom',age:28 };
p1.name = 'Sam' // 컴파일 에러
  • interface 키워드를 사용하여 타입을 정의한다.
  • ? 가 붙은 속성은 객체에서 없어도 되는 선택속성이다.
  • readonly 키워드를 이용해서 읽기 전용 속성으로 만들 수 있다. 읽기 전용은 말 그대로 변하지 않고 읽기만 가능하다.

# 인덱스 타입 정의하기

인터페이스에서 속성 이름을 구체적으로 정의하지 않고 값의 타입만 정의하는 것이 인덱스 타입

interface Person {
  readonly name: string;
  age: number;
  [key: string]: string | number;
}

# 인터페이스로 클래스 구현

interface Person {
  name: string;
  age: number;
}
class SomePerson implements Person {
  //...
}

implements 키워드를 사용해서 인터페이스를 클래스로 구현할 수 있다.

# 인터페이스 확장

interface Person {
  name: string;
  age: number;
}
interface Korean extends Person {
  isLiveInSeoul: boolean;
}
  • extends 키워드를 통해서 확장이 가능하다
  • 여러 개의 인터페이스도 확장이 가능하다

# 인터페이스의 타입 호환성

타입스크립트는 덕 타이핑(duck typing),구조적 타이핑(structural typing) 즉, 값이 가진 내부 구조에 기반해서 타입 호환성을 검사한다.

interface Person {
  name: string;
  age: number;
  //age?: number;
}
interface Product {
  name: string;
  age: number;
}
const person: Person = { name: 'Tom', age: 28 };
const product: Product = person;

만약 Person 의 age 가 선택속성이라면, Person 값의 집합이 Product 값의 집합보다 작아질 수 있어 할당이 불가능하다. 집합의 대소관계를 파악해야 한다.
